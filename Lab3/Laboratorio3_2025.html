Parte 2: Regresin Lineal

Encontrar los parmetros $\theta = \begin{pmatrix} \theta_{1} \\ \theta_{2} \end{pmatrix}$ solucin de $\underset{\theta}{\arg\min} \sum_{i=1}^{n}|Y_i-f_{\theta}(\mathbf{X}_i)|^2$ Donde $f_{\theta}(\mathbf{X}) = \theta_0 + \sum_{d=1}^{D}\theta_d X_d$ (en nuestro caso $D=2$)

Calcular el coeficiente $R^2$. $$R^2=1-\frac{\sum_{i=1}^{n}|Y_i-f(\mathbf{X}_i)|^2}{\sum_{i=1}^{n}|Y_i-\overline{Y}|^2}$$

Donde $\mathbf{X} = \begin{pmatrix} X_{1} \\ X_{2} \end{pmatrix}$ y $\overline{Y}=\frac{1}{n}\sum_{i=1}^{n}Y_i$

Parte 3: mejorar el modelo aplicando una transformacin bien elegida

El modelo inicial es: $Y = 3 X_1 - 2 X_2^2 + \epsilon$. Aadiremos coeficientes polinomiales con PolynomialFeatures.

El conjunto de descriptores inicial $\mathbf{X} = \begin{pmatrix} X_{1} \\ X_{2} \end{pmatrix}$ se convertir ahora en $\mathbf{X} = \begin{pmatrix} 1 \\ X_{1} \\ X_{2} \\ X_{1}^2 \\ X_{1}X_{2} \\ X_{2}^2 \end{pmatrix}$ lo que deja:

$$f_{\theta}(\mathbf{X}) = \theta'_0 + \theta_1 X_1 + \theta_2 X_2 + \theta_3 X_1^2 + \theta_4 X_1X_2 + \theta_5 X_2^2$$

Parte 7: Aumentar el nmero de caractersticas y regularizar una regresin

El objetivo es examinar el impacto de la regularizacin de los coeficientes de una regresin a medida que aumenta el nmero de coeficientes. Utilizamos caractersticas polinmicas y tcnicas de regularizacin Ridge o Lasso.

Regularizacin: agregar una penalizacin al modelo (a la funcin de entrenamiento) para evitar que aprenda demasiado los datos de entrenamiento.

  • Lasso (o L1): se penaliza la suma del valor absoluto de los coeficientes.
  • Ridge (o L2): se penaliza la suma de los valores cuadrados de los coeficientes.

  1. Genere una tabla donde para cada degree=1..20, se guarde:

    • nmero de coeficientes de $\theta$ (igual al nmero de columnas de $X$)
    • $R^2$ para cada modelo de regresin
    • norma del vector de coeficientes para cada modelo de regresin
    • cantidad de coeficientes no nulos para cada moelo de regresin

    Hint: Arme un dataframe con los datos que le parezcan relevantes para la actividad. Para ello, puede llevar el registro para cada degree en la lista res de diccionarios que contengan la informacin relevante de cada uno.

from sklearn.linear_model import Ridge, Lasso
import numpy.linalg as nplin
import numpy

def coef_non_nuls(coef):
    return sum(numpy.abs(coef) > 0.001)

res = []
for d in range(1, 21):
    poly = PolynomialFeatures(degree=d)
    Xtr = poly.fit_transform(X_train)
    reg = LinearRegression().fit(Xtr, y_train)
    rid = Ridge(alpha=0.1).fit(Xtr, y_train)
    las = Lasso(alpha=0.1, max_iter=10000).fit(Xtr, y_train)
    for model, name in zip([reg, rid, las], ["Linear", "Ridge", "Lasso"]):
        coef = model.coef_
        res.append({
            "degree": d,
            "modelo": name,
            "n_coef": len(coef),
            "R2": r2_score(y_test, model.predict(poly.transform(X_test))),
            "norma_coef": nplin.norm(coef),
            "coef_no_nulos": coef_non_nuls(coef)
        })
df_poly = pd.DataFrame(res)

Por qu el desempeo de la regresin lineal cae? Por qu los modelos Ridge y Lasso difieren en la cantidad de coeficientes no nulos?

Cuando usamos regresin lineal polinmica y subimos mucho el grado del polinomio:

  • El modelo se vuelve muy complejo.

  • Empieza a memorizar los datos de entrenamiento en lugar de aprender la tendencia real.

  • Se genera un sobreajuste y hace que el modelo funcione peor con datos nuevos (prueba).

Por otro lado, los modelos Ridge y Lasso usan regularizacin, que pone un castigo a los nmeros muy grandes en los coeficientes del modelo:

  • Ridge (L2): reduce los coeficientes grandes, pero casi nunca los hace cero.

  • Lasso (L1): puede reducir algunos coeficientes a cero, lo que equivale a ignorar algunas variables automticamente.

Por eso, con el mismo polinomio, Lasso suele terminar con menos variables activas que Ridge.
